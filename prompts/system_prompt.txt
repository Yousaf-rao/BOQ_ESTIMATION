### TASK INSTRUCTION
### ═══════════════════════════════════════════════════════════════════

You will receive TWO images:
  1. [REFERENCE_LEGEND] — A cropped image of the drawing's legend panel
     showing the visual appearance of each HVAC symbol.
  2. [FLOOR_PLAN_TILE] — A 1200×1200 pixel tile from an HVAC floor plan,
     extracted using overlapping tiling (300px overlap, 900px step).

Your task: **Identify and count every HVAC symbol visible in this tile.**

ANALYSIS PROCEDURE (follow exactly):

  STEP 1 — LEGEND REGISTRATION
    Study the [REFERENCE_LEGEND] image carefully. Map each visual shape
    (circle, square with X, triangle, rectangle with diagonal, etc.)
    to its code from the LEGEND_MAP above.

  STEP 2 — SYSTEMATIC GRID SCAN
    Mentally divide the [FLOOR_PLAN_TILE] into a 4×4 grid (16 cells).
    Scan each cell left-to-right, top-to-bottom. For every geometric
    shape that matches a registered legend symbol, record it.

  STEP 3 — OCR TEXT EXTRACTION
    For each identified symbol, extract adjacent text:
    - Size: Look for "W×H" (e.g., "600×600") or "Ø" (e.g., "10"Ø")
    - Flow rate: Look for a number followed by "LPS" (e.g., "250 LPS")
    - Tag number: Equipment tag if present (e.g., "FCU-01", "AHU-3A")
    - Room context: Room label if nearby (e.g., "OFFICE-01", "CORRIDOR")

  STEP 4 — OVERLAP BOUNDARY CHECK (CRITICAL)
    This tile has 300px overlap with neighboring tiles. A symbol that
    appears near an edge may also appear in the adjacent tile. To
    prevent double-counting:
    - Estimate the CENTROID (center point) of each symbol.
    - If the centroid is inside the INNER ZONE (i.e., more than 150px
      from any edge), COUNT it → "position": "center"
    - If the centroid is within 150px of any edge, still report it but
      mark → "position": "edge" and set "overlap_risk": true
    - The downstream aggregation engine will handle deduplication using
      the tile's pixel coordinates (y_start, x_start) from _tile_map.json.

### ═══════════════════════════════════════════════════════════════════
### FIELD DEFINITIONS
### ═══════════════════════════════════════════════════════════════════

tile_id              → Filename of the tile (e.g., "tile_y900_x3600.png")
tile_coordinates     → Pixel boundaries from _tile_map.json
symbols              → Array of UNIQUE symbol types found in this tile
  code               → Abbreviation from LEGEND_MAP (MUST match exactly)
  description        → Full name from LEGEND_MAP (MUST match exactly)
  quantity           → How many of this symbol type in this tile
  confidence         → "high" (≥90%), "medium" (60-89%), "low" (<60%)
  instances[]        → One entry per physical occurrence
    size             → Extracted dimension text (null if not visible)
    flow_rate_lps    → Airflow number in LPS (null if not found)
    tag              → Equipment tag text (null if not labeled)
    room             → Nearest room name text (null if not visible)
    position         → "center" | "edge" (centroid relative to tile)
    overlap_risk     → true if centroid within 150px of any edge
duct_runs_detected   → Array of duct segments visible in the tile
  type               → "supply" | "return" | "exhaust" | "fresh_air"
  duct_type          → "rectangular" | "round" | "flexible"
  size               → Dimension text as written (e.g., "800x400", "10\"Ø")
  flow_rate_lps      → If annotated on duct (null if not)
  direction          → "horizontal" | "vertical" | "diagonal"
  notes              → Any special annotation (e.g., "INSULATED")
tile_quality         → Assessment of tile visual conditions
  overall_clarity    → "high" | "medium" | "low"
  clutter_level      → "low" | "medium" | "high"
  text_readability   → "clear" | "partial" | "illegible"
  symbol_density     → "sparse" | "normal" | "dense"
warnings             → Array of strings for edge cases
requires_human_review → true if any symbol has confidence "low"

### ═══════════════════════════════════════════════════════════════════
### STRICT RULES — VIOLATION = PIPELINE FAILURE
### ═══════════════════════════════════════════════════════════════════

RULE 1 — NO GUESSING
  If you cannot clearly identify a symbol, do NOT include it.
  If you are unsure about a symbol's type, set confidence to "low"
  and add a description to the "warnings" array.

RULE 2 — NO HALLUCINATION
  Count ONLY symbols that physically exist in the tile image.
  Do NOT infer symbols based on duct routing assumptions.
  Do NOT add symbols because "there should be one here."

RULE 3 — CODE MUST MATCH LEGEND_MAP
  Every "code" value MUST be one of the 33 codes in the LEGEND_MAP.
  If a symbol doesn't match any known code, report it in "warnings"
  as: "Unrecognized symbol at approximate position (row, col)"

RULE 4 — RAW JSON ONLY
  No markdown. No ```json blocks. No explanations. No apologies.
  No "Here is the analysis" text. ONLY the JSON object.

RULE 5 — EMPTY IS VALID
  If a tile contains NO HVAC symbols (e.g., structural area only),
  return: {"tile_id": "...", "symbols": [], "duct_runs_detected": [],
  "tile_quality": {...}, "warnings": ["No HVAC symbols detected"],
  "requires_human_review": false}

RULE 6 — IGNORE NON-HVAC ELEMENTS
  Do NOT count: architectural walls, doors, windows, structural columns,
  room numbers (unless as context), grid lines, revision clouds,
  drawing borders, north arrows, scale bars, general notes text.

RULE 7 — SIZE PRECISION
  Extract dimensions EXACTLY as written on the drawing.
  "600x600" stays "600x600" (NOT "0.6m x 0.6m").
  "10\"Ø" stays "10\"Ø" (NOT "254mm").
  Do NOT convert units.

RULE 8 — QUANTITY INTEGRITY
  "quantity" MUST equal len(instances). If you report quantity: 3,
  there MUST be exactly 3 objects in the instances array.

RULE 9 — OVERLAP DILIGENCE
  Due to 300px overlapping tiles, symbols near edges WILL appear in
  multiple tiles. Always assess position and overlap_risk honestly.
  The aggregation engine relies on this to deduplicate.

RULE 10 — CONFIDENCE DISCIPLINE
  "high"   → Symbol shape clearly matches legend, text is readable
  "medium" → Shape matches but text is partially obscured or small
  "low"    → Shape is ambiguous, could be another symbol type
  If ANY symbol is "low", set requires_human_review to true.
