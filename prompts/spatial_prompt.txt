### ═══════════════════════════════════════════════════════════════════
### CRITICAL CONTEXT RULES — HOW TO DISTINGUISH SYMBOLS FROM NOISE
### ═══════════════════════════════════════════════════════════════════

Engineering drawings contain THOUSANDS of lines. Most are NOT symbols.
Use these CONTEXTUAL RULES to avoid false positives:

RULE A — SYMBOLS ATTACH TO DUCTS
  Real HVAC symbols (diffusers, dampers, valves) are ALWAYS connected
  to a duct line. If a shape is floating in empty space with no duct
  connection, it is NOT a symbol.

RULE B — WALLS ARE NOT SYMBOLS
  Wall corners, wall junctions, and structural lines look like
  rectangles and "L" shapes. These are NOT diffusers or dampers.
  Walls are thick parallel lines forming room boundaries.

RULE C — TEXT ANNOTATIONS ARE NOT SYMBOLS
  Numbers like "600x600", "250 LPS", room names like "OFFICE-01"
  are TEXT, not symbols. Read them as metadata for nearby symbols.

RULE D — SIZE MATTERS
  Real HVAC symbols in floor plans are small (roughly 15-40 pixels
  in a 1024x1024 tile). If a shape is too large (>100px), it's
  likely a room boundary or duct run, NOT a symbol.

RULE E — DUCT RUNS ARE NOT SYMBOLS
  Long parallel lines (horizontal or vertical) are duct runs.
  They carry air. Symbols are the small devices ATTACHED to these runs.

  Step 2: Divide the TILE into a 4×4 mental grid

### ═══════════════════════════════════════════════════════════════════
### FIELD DEFINITIONS
### ═══════════════════════════════════════════════════════════════════

label           → Symbol code from legend (MUST be valid code)
description     → Full name of the symbol
box_2d          → [ymin, xmin, ymax, xmax] in 0-1000 normalized range

confidence      → "high" (clearly visible), "medium" (partially obscured), "low" (uncertain)

### ═══════════════════════════════════════════════════════════════════
### STRICT RULES — ZERO TOLERANCE
### ═══════════════════════════════════════════════════════════════════

RULE 1 — NO GUESSING
  If you cannot clearly identify a symbol, DO NOT include it.
  Better to miss a real symbol than to report a false one.
  OpenCV gave 1977 false hits. You must give ZERO false hits.

RULE 2 — NO HALLUCINATION
  Count ONLY symbols that PHYSICALLY EXIST in the tile.
  Do NOT add symbols because "there should be one here".
  Do NOT count wall corners, duct junctions, or text as symbols.

RULE 3 — EMPTY IS VALID
  If a tile has NO HVAC symbols (just walls, structure, or empty
  space), return: {"tile_id": "...", "detections": [],
  "tile_summary": {"total_symbols_found": 0, "has_duct_runs": false,
  "tile_quality": "high"}}

RULE 4 — BOUNDING BOX PRECISION
  Each box must tightly wrap the symbol. Not too big, not too small.
  A good bounding box covers the symbol shape with ~5px margin.

RULE 5 — ONE BOX PER SYMBOL
  Each physical symbol gets exactly ONE bounding box.
  Do NOT create multiple boxes for parts of the same symbol.

RULE 6 — PURE JSON ONLY
  No markdown. No ```json. No explanation. ONLY the JSON object.
